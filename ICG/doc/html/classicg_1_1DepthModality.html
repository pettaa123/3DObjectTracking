Detailed Description

Modality class that implements a depth-based modality, which uses information from a DepthCamera and DepthModel to calculate the gradient vector and Hessian matrix that are used by the Optimizer to update the Body pose.

The modality is able to measure occlusions using the depth image or model occlusions using renderings from a FocusedDepthRenderer object.

Parameters
depth_camera_ptrreferenced DepthCamera from which depth images are taken.
depth_model_ptrreferenced DepthModel that provides views from a Sparse Viewpoint Model.
depth_renderer_ptrreferenced FocusedDepthRenderer that is used for modeled occlusion handling.
n_pointsnumber of correspondence points.
stride_lengthdistance in meter between points that are sampled during the correspondence search.
considered_distancesconsidered distance for each iteration. The distance defines the area in meter in which points are sampled during the correspondence search. If fewer values than n_corr_iterations are given, the last provided distance is used.
standard_deviationsuser-defined standard deviation for each iteration. If fewer values than n_corr_iterations are given, the last provided standard deviation is used.
measure_occlusionsdefines if occlusions are measured using the depth image.
measured_depth_offset_radiusradius in meter that specifies the depth offset provided by the DepthModel and that is used to measure occlusions.
measured_occlusion_radiusradius in meter that defines the area in which depth measurements from a DepthCamera are considered for occlusion handling.
measured_occlusion_thresholddefines how much smaller the minimum value from the DepthCamera is allowed to be compared to the value that is expected from the current estimate.
modeled_depth_offset_radiusradius in meter that specifies the depth offset provided by the DepthModel and that is used to model occlusions.
modeled_occlusion_radiusradius in meter that defines the area in which depth measurements from a FocusedDepthRenderer are considered for occlusion handling.
modeled_occlusion_thresholddefines how much smaller the minimum value from the FocusedDepthRenderer is allowed to be compared to the value that is expected from the current estimate.
n_unoccluded_iterationsnumber of iterations, after StartModality() is called, in which occlusion handling is turned off.
min_n_unoccluded_pointsminimum number of points that have to be valid. Otherwise, occlusion handling is turned off.
visualize_correspondences_correspondencevisualize correspondence points during VisualizeCorrespondences().
visualize_points_correspondencevisualize model points during VisualizeCorrespondences().
visualize_points_depth_rendering_correspondencevisualize model points on the depth rendering, which is used for occlusion handling, during VisualizeCorrespondences().
visualize_points_optimizationvisualize model points during VisualizeOptimization().
visualize_points_resultvisualize model points during VisualizeResults().
visualization_min_depthminimum depth for normalization of depth images in visualization.
visualization_max_depthmaximum depth for normalization of depth images in visualization.
