Detailed Description

Modality class that implements a region-based modality, which uses information from a ColorCamera and RegionModel to calculate the gradient vector and Hessian matrix that are used by the Optimizer to update the Body pose.

The modality is able to measure occlusions using images from a referenced DepthCamera that is close to the referenced ColorCamera and model occlusions using renderings from a FocusedDepthRenderer object.

Parameters
color_camera_ptrreferenced ColorCamera from which images are taken.
region_model_ptrreferenced RegionModel that provides views from a Sparse Viewpoint Model.
depth_camera_ptrreferenced DepthCamera that is used to for measured occlusion handling.
depth_renderer_ptrreferenced FocusedDepthRenderer that is used for modeled occlusion handling.
n_linesnumber of correspondence lines
min_continuous_distanceminimum distance in segments for which the foreground is not interrupted by the background and vice versa.
function_lengthtotal length in segments and number of discrete values considered by smoothed step functions.
distribution_lengthtotal length in segments and number of discrete values considered for the probability density function.
function_amplitudeamplitude parameter that models global uncertainty in smoothed step functions.
function_slopeslope parameter that models local uncertainty in smoothed step functions.
learning_ratefactor that specifies how far the optimization proceeds in the direction of a gradient calculated from finite differences.
n_global_iterationsnumber of global optimization iterations conducted before switching to local optimization.
scalesscale for each iteration. If fewer scales than n_corr_iterations are given, the last provided scale is used.
standard_deviationsuser-defined standard deviation for each iteration. If fewer values than n_corr_iterations are given, the last provided standard deviation is used.
n_histogram_binsnumber of bins that is used to discretize each dimension of the RGB color space. Has to be 2, 4, 8, 16, 32, or 64.
learning_rate_flearning rate that is used in the update of the foreground histogram.
learning_rate_blearning rate that is used in the update of the background histogram.
unconsidered_line_lengthdistance along the line from the center that is not considered in the update of color histograms.
max_considered_line_lengthmaximum length along the line that is considered in the update of color_histograms.
measured_depth_offset_radiusradius in meter that specifies the depth offset provided by the RegionModel and that is used to measure occlusions.
measured_occlusion_radiusradius in meter that defines the area in which depth measurements from a DepthCamera are considered for occlusion handling.
measured_occlusion_thresholddefines how much smaller the minimum value from the DepthCamera is allowed to be compared to the value that is expected from the current estimate.
modeled_depth_offset_radiusradius in meter that specifies the depth offset provided by the RegionModel and that is used to model occlusions.
modeled_occlusion_radiusradius in meter that defines the area in which depth measurements from a FocusedDepthRenderer are considered for occlusion handling.
modeled_occlusion_thresholddefines how much smaller the minimum value from the FocusedDepthRenderer is allowed to be compared to the value that is expected from the current estimate.
n_unoccluded_iterationsnumber of iterations, after StartModality() is called, in which occlusion handling is turned off.
min_n_unoccluded_linesminimum number of lines that have to be valid. Otherwise, occlusion handling is turned off.
visualize_lines_correspondencevisualize correspondence lines during VisualizeCorrespondences().
visualize_points_correspondencevisualize correspondence line centers during VisualizeCorrespondences().
visualize_points_depth_image_correspondencevisualize correspondence line centers on the depth image, which is used for occlusion handling, during VisualizeCorrespondences().
visualize_points_depth_rendering_correspondencevisualize correspondence line centers on the depth rendering, which is used for occlusion handling, during VisualizeCorrespondences().
visualize_points_optimizationvisualize correspondence line centers during VisualizeOptimization().
visualize_points_histogram_image_optimizationvisualize correspondence line centers on a histogram image, which shows pixel-wise posteriors for each pixel, during VisualizeOptimization().
visualize_points_resultvisualize correspondence line centers during VisualizeResults().
visualize_points_histogram_image_resultvisualize correspondence line centers on histograms images, which show pixel-wise posteriors for each pixel, during VisualizeResults().
visualization_min_depthminimum depth for normalization of depth images in visualization.
visualization_max_depthmaximum depth for normalization of depth images in visualization.
